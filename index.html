Shooting Game

by: Kazuki Girand

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make this game because it was similar enough to League Invaders that I had a basic picture in my head on how to create it but its different enough to be challenging. I also knew that trying to recreate gravity would be very complicated so I sticked to a basic shooting game.

2. What I Learned

I learned that I needed to explain everything in the instructions very carefully for the user to understand. I also learned that people like playing games with graphics a lot more than games without them.

3. Architecture of My Game

There is a class for each moving piece of the game along with everything behind the scenes. There are classes for the bullets, asteroids, stronger asteroids and the ship. There is also a game panel class along with a game objects class to keep the panels and collision boxes organized. Finally there is the object manager class that keeps the array lists organized.

4. Technical Implementation

There are array lists for the asteroids and bullets. There are variables like isActive, hits and collisionBox to make sure that any inactive bullets or asteroids are removed from their array lists. For the different panels of the game(menu, instructions, game, end) there is a int called currentState that represents each one of the panels. I am proud of the graphics in this game because it took a while to find the right graphics that had transparent backgrounds.

5. Most Challenging Part

I had a difficult time writing the code that organizes the variable that holds the number of times each asteroids has been hit. It took a while for the part that removed the asteroids from the array list when they were hit three times to work but eventually it did eventually.

6. Future Fixes and Updates

If I were to continue working on this game I would want to add levels and coins to buy upgrades and other things.
